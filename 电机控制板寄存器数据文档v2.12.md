## 1.引言

1个电机控制板控制1个电机，但是其共用同一个硬件电路板。在设计通信协议时，本文考虑到兼容性，所有不同电机共用一套通信协议。

通信协议采用基于RS485半双工通信的Modbus通信协议，所有电机控制板工作在从机模式，上位机充当主机。为了简化仅保留了保持寄存器，且全部电机共享下文列写的寄存器地址定义。因此，电机控制板只需要实现读`寄存器0x03`、`写单个寄存器0x06`和`写多个寄存器0x10`这3个功能码的应答及其异常帧即可。异常帧将功能码最高位置位来表示，即`0x83`、`0x86`和`0x90`。

针对不同功能的电机，如果尝试读写不支持的寄存器地址，需要返回对应的错误码给主机。同时，对于错误码的定义，本文不同于Modbus官方定义，本文会有专门章节用于叙述错误码含义。

**注意：下文全部数字默认为10进制，0x开头代表16进制，0b开头代表2进制**

## 2.寄存器定义

所有寄存器均为保持寄存器，不再区分线圈、输入寄存器等。地址编码从1开始，省略开头的4x。例如0001寄存器代表4x0001。

与标准Modbus有所区别，由于上位机和控制板均为小端存储CPU，为了加速运算，**所有寄存器数据均使用小端模式存储！**

例如某寄存器的值为`0x1234`，则发送数据时的字节顺序为`0x34 0x12`。
又如某32位数的值为`0x12345678`，则发送数据的字节顺序为`0x78 0x56 0x34 0x12`。

### 2.1 寄存器地址分区

为了方便区分不同功能类型的寄存器数据，在寄存器调整升级时，将一类功能的寄存器尽量归为连续内存区域，以提高读写性能。故将寄存器按照功能类型进行分开存储。

RS485寄存器地址最大支持65535个编址，故设计地址编址规律为：`00YXX`，其中Y代表地址分区（见下表），XX代表寄存器地址。

|  分区范围   |   功能   |                         描述                         |
| :---------: | :------: | :--------------------------------------------------: |
| 00000~00099 | 系统参数 |       只读寄存器居多，其余写入后需等重启后生效       |
| 00100~00199 | 运行参数 |    执行运动时通用的参数，修改后需等下次执行时生效    |
| 00200~00299 | 实时参数 |       电机运行时实时生效的参数，设置后立刻生效       |
| 00300~00399 | 原始数据 |  传感器、运行时flag、运转状态等数据/参数，只读居多   |
| 00400~00499 | 执行参数 | 命令电机做动作之前需要写入的参数以及执行状态、错误等 |
| 00500~00599 | 触发命令 |        对电机下具体命令触发用寄存器，均为只写        |
| 00600~00699 | 转发缓存 |                   用于存储转发数据                   |



### 2.2 寄存器地址定义

地址均为16进制，省略了前导0x。

读写中的R代表只读，W代表只写，RW代表可读可写

必要中打勾（√）的寄存器表示必须在本次版本实现，未打勾代表未将来扩展预留不必立刻实现。

| 圆盘存储 |     0=未知，1=对准，2=超过，3=未到     |                 0=原点，其余数字代表格口编号                 |

垂直升降机构的编码比较复杂，低8位位置判断流程为：

1）如果整个低8位就是0x00，代表原点

2）如果最高位为0，代表格口号，最多有127种组合（0x01~0x7F）

3）如果最高位为1，先检查是否在特殊位置内（0xFD~0xFF），也可用带符号数判断（-3~-1）；若是则代表卷入卷出口。

4）剩下的124种组合（0x80~0xFC）代表批量存入口。

## 3. 执行参数详解

### 3.1 执行任务号与参数解释

为了简化表示，下表中“参数”二字用“A”代替（例如“A1”代表“参数1”）：

| 任务ID |    任务名     | 参数 |                             解释                             |
| :----: | :-----------: | :--: | :----------------------------------------------------------: |
|  0x01  |   E2ROM操作   | 1个  | A1=1代表：将0~199地址的参数存入E2ROM中<br />A1=2代表：恢复出厂预置参数（需配合复位重启生效）<br />A1=3代表：将当前值存储为默认值 |
|  0x02  |    回零位     | 0个  | 执行回零操作，速度信息从109~112寄存器读取<br />A1=1代表电机1、A1=2代表电机2、A1=3代表夹爪 |
|  0x03  |   任务位移    | 3个  | 旋转或升降到A1低8位标识的位置，其值含义参见2.2.3节<br />A2和A3如果为0则忽略，否则A2A3共同组成32位整数以修正粗定位位置。（A2为低16位，A3为高16位）<br />A4代表是证件型还是卡片型，0=证件，1=卡片 |
|  0x04  |   原始位移    | 2个  | A1A2组成32位数表示立刻以当前位置为起始旋转这个数的位置。（A1为低16位，A2为高16位，32位带符号补码表示）<br />A3代表模式：1=相对位移、2=绝对位移<br />A4、A5、A6分别代表速度、加速度、减速度，若为0则使用内部值，否则覆盖配置 |
|  0x05  |     对准      | 1个  |      忽视位移，直接以当前位置试图对准A1低8位标识的位置       |
|  0x06  |     卷证      | 2个  | A1的值代表启动哪个卷证电机，A2代表正反转<br />A1：1=卷入卡，2=卷入证，3=卷出，4=批量口<br />A2：1=正转，2=反转<br />A3:手动控制延时时间，0=用内部值，非0=用A3的值（1~65535，单位：毫秒） |
|  0x07  | 伸缩/开合夹爪 | 1个  | A1：0x1000=闭合夹爪，0x2000=半开夹爪，0x3000=全开夹爪，0x4000=微开夹爪 |
|  0x08  |   转发数据    | 3个  | A1代表转发从设备：高8位为1时，低8位：1=电机1，2=电机2，3=夹爪；高8位为0时，低八位代表转发目标地址<br />A2：高8位为1时是写入/0为查询，低8位代表待读取/写入的寄存器个数<br />A3：代表转发到目标寄存器的起始地址<br />A4~A9：待转发数据/接收到的转发内容（最大3个寄存器，即6字节） |
|  0x09  |   出厂校准    | 1个  | 根据情况是否实现，若测试发现机械误差较大，则初始运动位置需要校准。 |
|  0x0A  |  脉冲式旋转   | 4个  | A1：脉冲频率低16位<br />A2：脉冲频率高16位<br />A3：占空比：0~1000，分别代表0%~100.0%<br />A4：旋转持续脉冲数：0~65535，0代表永续旋转 |
|  0x77  |   测试专用    | N个  |                        情况，用于测试                        |

“任务位移”与“原始位移”的区别：任务位移通过下达目标格口号/方向辅以可选的绝对位置修正值，控制板将执行包含对准的完整的定位流程，同时其会检查此时是否在零位；而原始位移模式将从当前位置直接进行相对位移，不会执行对准操作，并且起始时不会检查是否在零位。

| 5~7  |                             保留                             |
| 8~15 |               任务执行步骤号，主要用于内部调试               |

### 3.3 故障码

|  故障码   |            含义            |
| :-------: | :------------------------: |
|   0x00    |           无故障           |
|   0x01    | 执行器（电机/夹爪...）故障 |
|   0x02    |        任务执行超时        |
|   0x03    |        定位计数故障        |
|   0x04    |       对准/微调失败        |
|   0x05    |         回零位失败         |
|   0x06    |        不支持的任务        |
|   0x07    |          限位触发          |
|   0x10    |          用户急停          |
| 0x80~0xEF |       自定义其他故障       |
|   0xF0    |       其他未定义故障       |

## 4. Modbus错误码

| 错误码 |                  含义                  |
| :----: | :------------------------------------: |
|  0x01  |    非法功能,通常由于任务类型不支持     |
|  0x02  |             非法寄存器地址             |
|  0x04  |                设备故障                |
|  0x05  |     确认，表示任务已收到但执行较长     |
|  0x07  | 设备正忙，通常因为重复下达执行命令导致 |
|  0x08  |              CRC校验错误               |
|  0x0A  |                地址错误                |
|  0x0B  |              设备响应失败              |
